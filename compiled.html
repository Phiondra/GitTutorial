Erstellen und einbinden eines SSH-Keys

Erstellen eines Keys

  1. Terminal (Mac, Linux) oder Git Bash (Windows) öffnen 
  2. cd ~/.ssh
    Falls dieses Verzeichnis nicht existiert dieses erstellen 
    mkdir ~/.ssh
  3. Sicheren Sie ihre alten Keys
  4. Generieren eines neuen Keys 
    ssh-keygen -t rsa -C "your_email@youremail.com"
    Dann Speicherort angeben oder einfach Enter drücken. Anschliessend ein Passwort eingeben.

Erstellen und einbinden eines SSH-Keys

Einbinden eines Keys

  1. Bei GitHub zu "Einstellungen" - "SSH Keys"
  2. Im .ssh Verzeichnis den Key öffnen
    WICHTIG: den .pub Schlüssel öffnen
  3. Den ganzen Inhalt des Schlüssels kopieren und auf GitHub einfügen

Informationen für Git einstellen

  • Den Namen eingeben 
    git config --global user.name "Firstname Lastname"
  • Die Mail-Adresse eingeben 
    git config --global user.email "your_email@youremail.com"

Repository

Erstellen eines Repositorys

  • GitHub - new Repository
  • Name (Bsp: Hello-World) angeben, wenn gewünscht auch noch Beschreibung und Homepage
  • Erstellen eines Verzeichnisses mit dem Namen des Repos und in dieses wechseln 
    mkdir ~/Hello-World
cd ~/Hello-World
  • Git initialisieren 
    git init
  • Herkunft für das Repository hinzufügen (username ist Ihr Benutzernamen) 
    git remote add origin git@github.com:username/Hello-World.git

Repository

Ändern eines Repositorys

  • README-File erstellen 
    touch README
  • README-File für das Repository bereitstellen 
    git add README
  • Alle bereitgestellten Änderungen lokal hinzufügen (Nachricht ist "first commit") 
    git commit -m 'first commit'
  • Den Commit dem Richtigen Repository hinzufügen ("-u origin master" ist nur bei mehreren Branches nötig) 
    git push -u origin master
